### Vector DB Use Case

A traditional keyword based database search would not be sufficient for a system where lawyers search through 500-page contracts using natural language questions. Keyword search relies on exact word matching, which fails when the query and the document use different terminology. For example, a lawyer might search for "termination clauses," but the contract could use phrases like "agreement cancellation" or "end of contract." In such cases, keyword-based systems may miss relevant results or return incomplete information.
A vector database, on the other hand, enables semantic search by converting both documents and queries into numerical embeddings. These embeddings capture the meaning and context of the text rather than just exact words. When a lawyer asks a question in plain English, the system converts the query into an embedding and compares it with embeddings of contract sections using similarity measures such as cosine similarity.
This approach allows the system to retrieve the most relevant sections even if the wording differs. Additionally, vector databases are optimized for fast similarity searches across large datasets, making them suitable for handling lengthy legal documents. Therefore, a vector database plays a crucial role in enabling accurate, context-aware, and efficient search in such a system.
